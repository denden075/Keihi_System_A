# 要件定義書：プロジェクト費用管理アプリ

**バージョン：** 1.0  
**作成日：** 2026-04-19

---

## 1. 概要

会社のプロジェクトを一覧管理し、各プロジェクトに紐づく費用を記録・集計するWebアプリケーション。  
ローカル環境でのシングルユーザー運用を想定。

---

## 2. 技術スタック

| 要素 | 技術 |
|------|------|
| フロントエンド | HTML / CSS（Bootstrap） / JavaScript |
| バックエンド | Python（Flask） |
| データベース | SQLite |
| ORM | SQLAlchemy |

## 3. 機能要件

### 3-1. プロジェクト管理

| # | 機能 | 詳細 |
|---|------|------|
| P1 | プロジェクト登録 | 名称・説明・独自採番・注文書番号・開始日・終了日（任意）・ステータス・予算上限・売上金額・検収年月を登録 |
| P2 | プロジェクト一覧 | 一覧表示、ステータスでフィルタ可能、費用合計・予算残高・純売りを表示 |
| P3 | プロジェクト編集 | 登録情報の変更（予算上限・売上金額・検収年月の変更含む） |
| P4 | プロジェクト削除 | 紐づく費用ごと削除（確認ダイアログあり） |

**プロジェクトのステータス：** 見積中 / 進行中 / 完了 / 失注

**採番・売上項目：**
- 独自採番：社内管理用の任意番号（任意）
- 注文書番号：顧客から発行される注文書の番号（任意）
- 売上金額：プロジェクトの売上金額・円（任意）
- 純売り：売上金額 − 費用合計（自動計算）
- 検収年/月：売上を計上する年・月（任意）

**予算上限：**
- 登録時に任意で設定（未設定も可）
- 後から変更可能
- 費用合計が予算上限を超過した場合、一覧・詳細画面で警告表示

### 3-2. 費用管理

| # | 機能 | 詳細 |
|---|------|------|
| C1 | 費用登録 | プロジェクトに紐づけて費用を登録 |
| C2 | 費用一覧 | プロジェクト別に費用を一覧表示（発生日降順） |
| C3 | 費用編集 | 登録済み費用の変更 |
| C4 | 費用削除 | 個別削除（確認ダイアログあり） |
| C5 | CSV出力 | プロジェクト別の費用データをCSVエクスポート |

**費用の登録項目：**

| 項目 | 必須 | 詳細 |
|------|------|------|
| 費用名 | ○ | 例：外注費、交通費 |
| カテゴリ | ○ | 人件費 / 外注費 / 購入品 / 備品 / 交通費 / その他 |
| 金額（円） | ○ | 正の整数 |
| 発行日 | ○ | 請求書・領収書の発行日 |
| 領収日 | - | 実際の領収・受領日（発行日と異なる場合に入力） |
| 清算状況 | ○ | 未清算 / 清算済み |
| 備考 | - | 自由記述 |

### 3-3. 集計・レポート

| # | 機能 | 詳細 |
|---|------|------|
| R1 | プロジェクト別合計 | 各プロジェクトの費用合計・予算残高・売上・純売りを表示 |
| R2 | カテゴリ別集計 | プロジェクト内のカテゴリ別金額・割合 |
| R3 | 期間フィルタ | 発行日の期間を指定して絞り込み |
| R4 | ダッシュボード | 総費用・売上・純売り・進行中件数・予算超過件数、月別売上/純売り集計（3月決算・年度切替可） |

## 4. データモデル

```
projects
  id              INTEGER  PRIMARY KEY AUTOINCREMENT
  name            TEXT     NOT NULL
  description     TEXT
  project_number  TEXT                  -- 独自採番（任意）
  order_number    TEXT                  -- 注文書番号（任意）
  start_date      DATE     NOT NULL
  end_date        DATE
  status          TEXT     NOT NULL     -- '見積中' | '進行中' | '完了' | '失注'
  budget          INTEGER               -- 予算上限（円）、NULL=未設定
  sales_amount    INTEGER               -- 売上金額（円）、NULL=未設定
  accepted_year   INTEGER               -- 検収年（例: 2026）
  accepted_month  INTEGER               -- 検収月（1〜12）
  created_at      DATETIME NOT NULL
  updated_at      DATETIME NOT NULL

expenses
  id              INTEGER  PRIMARY KEY AUTOINCREMENT
  project_id      INTEGER  NOT NULL REFERENCES projects(id)
  name            TEXT     NOT NULL
  category        TEXT     NOT NULL     -- '人件費'|'外注費'|'購入品'|'備品'|'交通費'|'その他'
  amount          INTEGER  NOT NULL     -- 円
  issued_at       DATE     NOT NULL     -- 発行日
  received_at     DATE                  -- 領収日（任意）
  settlement      TEXT     NOT NULL     -- '未清算' | '清算済み'
  note            TEXT
  created_at      DATETIME NOT NULL
  updated_at      DATETIME NOT NULL
```

## 5. 画面構成

```
/ (ダッシュボード)
  - 全プロジェクト数・進行中件数
  - 総費用合計
  - 予算超過プロジェクト数（警告）
  - プロジェクト別費用ランキング（上位5件）

/projects (プロジェクト一覧)
  - ステータスフィルタ
  - 各プロジェクト：名称・ステータス・費用合計・予算上限・残高・超過警告

/projects/new (プロジェクト登録)
  - 登録フォーム

/projects/<id> (プロジェクト詳細)
  - プロジェクト情報・予算進捗バー
  - カテゴリ別集計
  - 期間フィルタ付き費用一覧
  - CSV出力ボタン

/projects/<id>/edit (プロジェクト編集)
  - 編集フォーム（予算上限変更含む）

/projects/<id>/expenses/new (費用登録)
  - 費用登録フォーム

/projects/<id>/expenses/<eid>/edit (費用編集)
  - 費用編集フォーム
```
